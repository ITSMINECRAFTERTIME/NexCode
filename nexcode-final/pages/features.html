Protection engine

Defense in depth

Each layer compounds. An attacker has to defeat all of them — and the layout changes with every compile.

🧬

Custom VM per build

Randomized opcode mappings and handler structures. Cracking one build teaches nothing about the next.

🔀

Control flow scrambling

Execution paths are flattened and reordered so static analysis can't follow the logic.

🔐

String encryption

Every literal is encrypted and only decrypted at runtime, in memory, at the moment of use.

💣

Anti-decompiler injection

Malformed constructs crash or hang popular decompilers like unluac before they extract anything.

🎭

Junk & decoy logic

Optional dead code paths mislead anyone attempting to trace real behavior.

🛡️

Anti-tamper hooks

Runtime checks detect debuggers, hooks, and environment tampering — and can trigger a kill switch.

NXC_ Macros

Surgical compile-time control

Directives you drop into your source. They run as harmless stubs locally and transform at compile time — zero runtime overhead.

NXC_ENCFUNC(fn)Wraps a function so its body is encrypted with 256-bit protection and only reconstructed when called.
NXC_ENCSTR(str)Encrypts a string literal; decrypted at runtime in memory only, never stored in plaintext.
NXC_NO_VIRT(fn)Marks a hot path to skip VM virtualization, preserving native performance on critical loops.
NXC_JUNK()Injects context-aware decoy logic at this location to mislead reverse engineers.
NXC_SEND_WEBHOOK(url, data)Sends a Discord/HTTP webhook through an encrypted proxy so endpoints can't be extracted.
NXC_INIT()Pre-authentication hook — runs environment and integrity checks before your logic executes.
Key & license system

Licensing that does it all

Five key types, hardware locking, floating seats, and instant control. Everything a modern software business needs.

Day-locked

N days

Expires a set number of days after redemption. Perfect for rentals and time-boxed access.

Lifetime

Forever

One-time purchase, never expires. HWID-locked to prevent sharing.

Trial

Limited

Time-limited and execution-capped so users can evaluate before buying.

Subscription

Recurring

Auto-renewing access tied to your billing webhooks. Lapses instantly on non-payment.

FFA / Keyless

Free

Anyone can run the script; premium features gate behind a valid key check.

Floating seats

Pooled

A pool of N concurrent activations shared across a team or customer.

🔗

HWID locking

Keys bind to hardware on first run. Sharing fails silently on mismatched machines.

⏸️

Offline grace

Cached authentication lets legitimate users keep working through brief connectivity drops.

🛑

Kill switch

Disable any key, script, or user instantly — the change propagates on the next auth check.
