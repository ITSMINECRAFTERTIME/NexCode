Quick start

Get your first script protected in under five minutes.

NexCode works by adding NXC_ macros to your Lua source. Locally they behave as harmless passthrough stubs, so your code runs unchanged during development. At compile time, they transform into real protection.

1. Add the stubs

Drop this at the top of your script so it runs locally without NexCode:

-- Passthrough stubs — stripped & replaced at compile time if not NXC_OBFUSCATED then NXC_ENCFUNC = function(f) return f end NXC_ENCSTR = function(s) return s end NXC_JUNK = function() end end

2. Wrap what matters

local core = NXC_ENCFUNC(function() local url = NXC_ENCSTR("https://api.example.com/data") print("Running protected logic") end) core()

3. Compile & distribute

Upload through the dashboard or call the API. You'll get a protected build plus a key to hand to your users.

💡Keep your stubs in your source permanently. They're free locally and get replaced automatically at compile time — so one file works everywhere.

Macros

Compile-time directives for surgical control over protection.

MacroPurpose
NXC_ENCFUNC(fn)Encrypt a function body; reconstructed only when called
NXC_ENCSTR(s)Encrypt a string; decrypted in memory at use
NXC_NO_VIRT(fn)Skip VM virtualization on a hot path for performance
NXC_JUNK()Inject decoy logic at this location
NXC_SEND_WEBHOOK(u,d)Send a webhook via encrypted proxy
NXC_INIT()Pre-auth hook for integrity checks
⚡Use NXC_NO_VIRT on render loops and physics steps. Virtualization is powerful but adds overhead — skip it where every millisecond counts.

Compiling

Turn your source into a protected build.

You can compile two ways: through the dashboard's visual configuration panel, or programmatically via the API. Both give you the same protection options — VM virtualization, control flow scrambling, string encryption, anti-decompiler injection, junk code, and anti-tamper hooks.

Pick an optimization profile to balance strength against runtime cost:

  • Fast — lighter protection, minimal performance impact
  • Balanced — strong protection with reasonable overhead (recommended)
  • Max — maximum protection for high-value scripts
# Compile via the API curl -X POST https://api.nexcode.io/v1/scripts/scr_abc/compile \ -H "Authorization: Bearer nxc_sk_..." \ -d '{ "profile": "balanced", "vm": true, "encrypt_strings": true }'

Create a key

curl -X POST https://api.nexcode.io/v1/keys \ -H "Authorization: Bearer nxc_sk_..." \ -H "Content-Type: application/json" \ -d '{ "script_id": "scr_abc", "type": "day_locked", "duration_days": 30 }'
🔑Scope your API keys with permissions like keys:write or scripts:read so each integration only gets what it needs.

Webhooks

Get notified in real time when things happen.

NexCode sends signed webhooks for 10 event types including key.created, key.validated, key.revoked, key.hwid_reset, and validation.failed. Always verify the signature before trusting a payload.

Verifying signatures (Node.js)

const crypto = require('crypto'); function verify(body, sig, ts, secret) { const expected = 'sha256=' + crypto.createHmac('sha256', secret) .update(`${ts}.${body}`) .digest('hex'); return crypto.timingSafeEqual( Buffer.from(expected), Buffer.from(sig) ); }

Verifying signatures (Python)

import hmac, hashlib def verify(body, sig, ts, secret): expected = "sha256=" + hmac.new( secret.encode(), f"{ts}.{body}".encode(), hashlib.sha256 ).hexdigest() return hmac.compare_digest(expected, sig)
